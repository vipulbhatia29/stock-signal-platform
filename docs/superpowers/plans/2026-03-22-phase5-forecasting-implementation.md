# Phase 5: Forecasting, Evaluation & Background Automation — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a self-healing nightly pipeline that pre-computes signals, trains Prophet forecasts, evaluates past recommendations, detects model drift, and surfaces insights via agent tools + frontend UI.

**Architecture:** Backend-first. DB models → pipeline infra → Prophet engine → evaluation loops → API endpoints → agent tools → frontend. Each chunk produces testable, committable work.

**Tech Stack:** FastAPI, async SQLAlchemy, TimescaleDB, Celery + Redis, Prophet, pandas/numpy, Next.js, TanStack Query, shadcn Dialog/Popover, framer-motion

**Spec:** `docs/superpowers/specs/2026-03-22-phase5-forecasting-design.md`
**JIRA Epic:** KAN-106 (Phase 5 — Forecasting & Automation)

  - For each: look up StockPrice on target_date, fill actual_price + error_pct
  - Update ModelVersion.metrics with rolling MAPE (last 10 evaluations)
  - Track via PipelineRunner

### Task 15: Drift detection

**Files:**
- Modify: `backend/tasks/evaluation.py` (append)

- [ ] `check_drift()` — called after forecast_evaluation:
  - **Model drift:** If rolling MAPE > 20% → set ModelVersion.status = "degraded", queue `retrain_single_ticker`
  - **Volatility spike:** If 5-day vol > 2× 90-day avg → queue retrain
  - **VIX regime:** Fetch ^VIX via yfinance. If > 30 → set regime flag (stored in PipelineRun metadata)
- [ ] VIX fetch uses `with_retry()` helper, cached for pipeline run duration

### Task 16: Recommendation evaluation task

**Files:**
- Modify: `backend/tasks/evaluation.py` (append)
