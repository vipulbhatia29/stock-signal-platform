# Phase 8.5: Portfolio Analytics Upgrade — Specification

**Epic:** KAN-246 | **Stories:** KAN-247 (QuantStats), KAN-248 (PyPortfolioOpt), KAN-249 (pandas-ta)
**Date:** 2026-04-01 | **Status:** Reviewed & Approved (codebase-verified)
**Alembic head:** `2146d203aa47` (migration 021) — next migration is 022

## Goal

Replace hand-rolled portfolio analytics with battle-tested financial libraries (QuantStats, PyPortfolioOpt, pandas-ta) to improve accuracy, maintainability, and metric depth for part-time passive investors.

## Architectural Constraints

> **Materialize everything.** All computed metrics must be persisted to the database during pipeline runs. API endpoints read materialized data — never compute on-the-fly. Only exception: scenarios that depend on request-time context (future scope).

> **Keep scoring engine ours.** Neither QuantStats nor PyPortfolioOpt has portfolio scoring/grading. Our `compute_portfolio_health()` weighted composite (0-10 + letter grade) is the right approach. Improve the *inputs* it reads; don't outsource the opinion.

## Current State

| Area | Today | Replacement |
|------|-------|-------------|
| RSI, MACD, SMA, Bollinger | Hand-rolled (numpy/pandas) | pandas-ta (already in pyproject.toml, unused) |
| Sharpe, Volatility, Annual Return | Hand-rolled in `compute_risk_return()` | QuantStats (validated) |
| Sortino, Max Drawdown, Calmar, Alpha/Beta | **Not computed** | QuantStats (new) |
| Rebalancing | Equal-weight in `calculate_position_size()` | PyPortfolioOpt (efficient frontier) |
| Portfolio Health scoring | Weighted composite (5 components, 0-10) | Keep — enhance risk component inputs |

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| pandas-ta scope | Drop-in replace 4 existing indicators only | Users are passive investors, don't need Stochastic RSI/ADX/OBV |
| QuantStats scope | Portfolio-level + per-stock metrics | Dashboard health card + stock detail page |
| QuantStats HTML tearsheet | Skip | Not SaaS-worthy — just compute the numbers |
| Rebalancing default | Min Volatility | Research: min-vol produces stable weights, fewer trades, better for passive investors |
| Rebalancing options | Min Vol + Max Sharpe + Risk Parity | User selects via preferences |
| Composite score formula | Keep unchanged this phase | Don't mix library swap with formula changes — separate ticket |
| Health scoring risk component | Enhance inputs (add Sortino, max_drawdown) | Keep weights unchanged; better data in, better score out |
| Data threshold | Graceful degradation | Show available metrics with `data_days` field, null when < 30 days |
| Benchmark | SPY (hardcoded) | Configurable later |
| Materialization | All metrics stored in DB | Computed during nightly pipeline, API reads only |

### FR-2: QuantStats Portfolio & Stock Metrics (S1 — KAN-247)

**Per-stock metrics (materialized in `signal_snapshots` new columns):**
- Sortino ratio, max drawdown (%), alpha, beta (vs SPY)
- Computed from `stock_prices` daily close series (1y lookback) during nightly pipeline Phase 1
- New function: `compute_quantstats_stock(ticker, closes, spy_closes) -> dict`
- Called alongside `compute_signals()` in `nightly_price_refresh_task()`
- `store_signal_snapshot()` updated to write 4 new columns

**Portfolio metrics (materialized in `portfolio_snapshots` new columns):**
- CAGR, Sharpe, Sortino, max drawdown (%), max drawdown duration (days), Calmar, alpha, beta, VaR (95%), data_days
- Computed from portfolio snapshot value series during nightly pipeline Phase 4
- New function: `compute_quantstats_portfolio(portfolio_id, db) -> dict`
- Called in `snapshot_health_task()` after `snapshot_portfolio_value()`
- `snapshot_portfolio_value()` updated to write new columns (or separate function writes them)

**New API endpoints (read-only from DB):**
- `GET /api/v1/portfolio/analytics` → `PortfolioAnalyticsResponse` (reads latest `portfolio_snapshots` row)
- `GET /api/v1/stocks/{ticker}/analytics` → `StockAnalyticsResponse` (reads latest `signal_snapshots` row)
- Both include `data_days: int` field for transparency

**Health scoring enhancement:**
- `compute_portfolio_health()` risk component: incorporate materialized Sortino + max_drawdown from `signal_snapshots`
- Keep component weights unchanged (risk = 0.20)
- Better inputs → better score, no formula change

**New agent tool:**
- `PortfolioAnalyticsTool` — reads materialized QuantStats data from DB
- Does NOT modify existing `PortfolioHealthTool`

### FR-3: PyPortfolioOpt Rebalancing (S2 — KAN-248)

**New ORM model + table:**
- `RebalancingSuggestionModel` in `backend/models/portfolio.py`
- Table `rebalancing_suggestions` (see schema above)

**Materialized computation:**
- New function: `compute_rebalancing(portfolio_id, strategy, db) -> list[RebalancingSuggestionModel]`
- Called in nightly pipeline (new Phase 4 parallel task)
- Also triggered on portfolio change (after transaction logged)
- Uses PyPortfolioOpt: `EfficientFrontier(mu, S)` with strategy dispatch

**Three strategies via `UserPreference.rebalancing_strategy`:**
- `min_volatility` (default) — `ef.min_volatility()`
- `max_sharpe` — `ef.max_sharpe()`
- `risk_parity` — `HRPOpt(returns).optimize()` (Hierarchical Risk Parity)

**Input:** Current positions + 1y historical price matrix (daily returns from `stock_prices`)

**Constraints:**
- `ef.add_constraint(lambda w: w <= max_position_pct)` — from `UserPreference.max_position_pct` (default 5%)
- Sector caps from `UserPreference.max_sector_pct` (default 30%)
- Falls back to equal-weight if optimization fails (< 30 days data, singular covariance matrix, solver error)

**Existing endpoint enhanced:**
- `GET /api/v1/portfolio/rebalancing` reads from `rebalancing_suggestions` table
- Response shape: backward-compatible `RebalancingResponse` (existing Pydantic schema works)

**Preference column:**
- Add `rebalancing_strategy` VARCHAR(20) to `user_preferences`, default `'min_volatility'`
- New endpoint: `PATCH /api/v1/preferences` already exists — just validate new field

## User Stories

**US-1:** As a passive investor, I want to see Sortino, max drawdown, Calmar, alpha/beta on my portfolio so I understand risk-adjusted performance beyond Sharpe.
- AC: `GET /portfolio/analytics` returns materialized metrics from `portfolio_snapshots`
- AC: Returns null for metrics when < 30 days of snapshots, with `data_days` field

**US-2:** As a passive investor, I want per-stock Sortino, drawdown, alpha/beta so I can evaluate individual holdings.
- AC: `GET /stocks/{ticker}/analytics` returns materialized metrics from `signal_snapshots`
- AC: Uses SPY as benchmark

**US-3:** As a passive investor, I want optimized rebalancing that minimizes volatility.
- AC: `GET /portfolio/rebalancing` reads materialized suggestions from `rebalancing_suggestions`
- AC: Default strategy is min_volatility, selectable via preferences
- AC: Falls back to equal-weight if optimization fails

**US-4:** As a developer, I want pandas-ta for signal computation to eliminate hand-rolled indicator code.
- AC: RSI, MACD, SMA, Bollinger use pandas-ta internally
- AC: `SignalResult` output unchanged (19 fields, same types)
- AC: Existing signal tests pass within ±5% tolerance
