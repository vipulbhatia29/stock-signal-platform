- I prefer Python with type hints and Google-style docstrings
- I use `uv` for Python package management, not pip or poetry
- I use VS Code as my editor
- I prefer pytest for all Python testing

- Always include error handling — never let exceptions pass silently
- Prefer explicit over implicit
- Add logging with `logging.getLogger(__name__)`, not print statements

## When explaining things

- Explain the "why" alongside the "how" when introducing new concepts
